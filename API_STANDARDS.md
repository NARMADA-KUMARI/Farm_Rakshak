# FarmRakshak — API Standards

> **Version:** 1.0 | **Date:** 2026-03-26

---

## URL Convention

```
/api/v1/{resource}
/api/v1/{resource}/{id}
/api/v1/{resource}/{id}/{sub-resource}
```

Versioning is URL-based. No breaking changes within the same version.

---

## Response Envelope

### Success
```json
{
  "success": true,
  "data": { },
  "message": "Operation completed successfully",
  "timestamp": "2026-03-26T10:00:00Z",
  "traceId": "abc-123-def"
}
```

### Error
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid input",
    "details": [
      { "field": "email", "message": "must be a valid email" }
    ]
  },
  "timestamp": "2026-03-26T10:00:00Z",
  "traceId": "abc-123-def"
}
```

### Paginated
```json
{
  "success": true,
  "data": {
    "content": [],
    "page": 0,
    "size": 10,
    "totalElements": 100,
    "totalPages": 10
  },
  "timestamp": "2026-03-26T10:00:00Z"
}
```

---

## HTTP Status Codes

| Code | Usage |
|------|-------|
| 200 | Success |
| 201 | Created |
| 202 | Accepted (async processing) |
| 400 | Bad request / validation error |
| 401 | Unauthorized |
| 403 | Forbidden |
| 404 | Not found |
| 409 | Conflict (duplicate) |
| 429 | Too many requests |
| 500 | Internal server error |
| 503 | Service unavailable |

---

## Headers

| Header | Purpose |
|--------|---------|
| `Authorization: Bearer {token}` | JWT auth |
| `X-Trace-Id` | Correlation ID (generated by gateway) |
| `Accept-Language` | Locale for i18n responses |
| `Content-Type: application/json` | Standard payload type |

## Pagination Parameters

| Param | Default | Max |
|-------|---------|-----|
| `page` | 0 | — |
| `size` | 10 | 50 |
| `sort` | `createdAt,desc` | — |

## Naming Conventions

| Element | Convention | Example |
|---------|-----------|---------|
| Endpoints | kebab-case nouns | `/api/v1/crop-analyses` |
| JSON fields | camelCase | `diseaseName` |
| Query params | camelCase | `?pageSize=10` |
| Enums | UPPER_SNAKE | `PROCESSING`, `COMPLETED` |
